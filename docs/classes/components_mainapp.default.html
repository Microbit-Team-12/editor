If set, this.context will be set at runtime to the current value of the given Context.

Usage:

type MyContext = number
const Ctx = React.createContext<MyContext>(0)

class Foo extends React.Component {
  static contextType = Ctx
  context!: React.ContextType<typeof Ctx>
  render () {
    return <>My context's value: {this.context}</>;

componentWillUnmount

  • componentWillUnmount(): void

  • Overrides React.Component.componentWillUnmount

    • Defined in src/components/MainApp.tsx:188
    summary

    Disconnect the micro:bit before unmounting.

    For dev purposes only: upon recompilation, this component loses access to the {@link MainAppState.connection} object; if the web serial connection is not terminated here, the page must needs to be refreshed to reclaim the micro:bit interface.

    In production, {@link MainApp} is intended to be the top-level component to be dismounted only when the page is closed, after which the web serial connection is terminated anyways, making it unnecessary to manually invoke the disconnection procedure.

    Returns void

renderHeaderButton

  • renderHeaderButton(text: string, callback: () => void, isEnabled: () => boolean): Element
    • Defined in src/components/MainApp.tsx:209
    summary

    Renders a button in the header of the app.

    The 'Start' button is enabled only when {@link MainAppState.connection} is null, as it is intended to be used when:

    • the user first opens the page, and connects the micro:bit
    • the micro:bit is disconnected, and the user wants to reconnect it (or to connect another one)

    MicrobitState.Busy and MicrobitState.Free document when the other micro:bit related buttons are enabled. In addition, since the 'Flash' and 'Run' buttons in the header requires access to the code in the editor, they are enabled only when isEditorMounted returns true.

    The duck 'Help' button is always enabled.
